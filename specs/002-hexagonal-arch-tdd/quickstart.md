## Prerequisites

- Node.js 20+ with pnpm 9+
- Rust 1.75+ with cargo

## Quick Commands

```bash
# Install dependencies
pnpm install
cargo build --manifest-path src-tauri/Cargo.toml

# Development
pnpm tauri dev

# Run all checks (pre-commit equivalent)
./scripts/pre-commit.sh

# Run tests
pnpm test:run                                      # Frontend tests (114 tests)
cargo test --features test-mocks                   # Backend tests (174 tests)

# Run only domain tests (should complete in <5s)
cargo test domain:: --features test-mocks --lib

# Run architecture boundary tests
pnpm test:run -- --grep "Boundaries"

# Build production
pnpm build                   # Frontend typecheck + build
cargo build --release --manifest-path src-tauri/Cargo.toml
```

## Key Patterns

### 1. Defining a Port (Interface)

**TypeScript** (`src/ports/document-repository.port.ts`):
```typescript
export interface DocumentRepositoryPort {
  add(filePath: string, title?: string): Promise<Document>;
  getById(id: string): Promise<Document | null>;
  list(orderBy?: OrderBy): Promise<Document[]>;
}
```

**Rust** (`src-tauri/src/ports/document_repository.rs`):
```rust
#[cfg_attr(any(test, feature = "test-mocks"), automock)]
#[async_trait]
pub trait DocumentRepository: Send + Sync {
    async fn add(&self, file_path: String, title: Option<String>) -> Result<Document, DomainError>;
    async fn get_by_id(&self, id: String) -> Result<Option<Document>, DomainError>;
}
```

### 2. Implementing an Adapter

**TypeScript** (`src/adapters/tauri/document-repository.adapter.ts`):
```typescript
import { commands } from '../../lib/bindings';
import type { DocumentRepositoryPort } from '../../ports/document-repository.port';

export class TauriDocumentRepository implements DocumentRepositoryPort {
  async add(filePath: string, title?: string) {
    const result = await commands.libraryAddDocument(filePath, title ?? null, null);
    if (result.status === 'error') throw new Error(result.error);
    return result.data;
  }

  async getById(id: string) {
    const result = await commands.libraryGetDocument(id);
    if (result.status === 'error') throw new Error(result.error);
    return result.data;
  }
}
```

**Rust** (`src-tauri/src/adapters/sqlite/document_repo.rs`):
```rust
pub struct SqliteDocumentRepo {
    pool: Pool<Sqlite>,
}

#[async_trait]
impl DocumentRepository for SqliteDocumentRepo {
    async fn add(&self, file_path: String, title: Option<String>) -> Result<Document, DomainError> {
        sqlx::query_as!(...)
            .fetch_one(&self.pool)
            .await
            .map_err(|e| DomainError::storage(e.to_string()))
    }
}
```

### 3. Creating an Application Service

**Rust**:
```rust
pub struct LibraryService<R: DocumentRepository> {
    repository: R,
}

impl<R: DocumentRepository> LibraryService<R> {
    pub fn new(repository: R) -> Self {
        Self { repository }
    }

    pub async fn add_document(&self, path: String, title: Option<String>) -> Result<Document, DomainError> {
        // Business logic: validate, hash file, etc.
        self.repository.add(path, title).await
    }
}
```

### 4. Thin Tauri Command Handler

```rust
#[tauri::command]
#[specta::specta]
pub async fn library_add_document(
    db: State<'_, DbInstances>,
    file_path: String,
    title: Option<String>,
) -> Result<Document, String> {
    let service = create_library_service(&db).await?;
    service.add_document(file_path, title).await.map_err(|e| e.to_string())
}
```

### 5. Dependency Injection in React

```typescript
// src/adapters/context/repository-context.tsx
import { createContext, useContext } from 'react';
import { TauriDocumentRepository } from '../tauri/document-repository.adapter';

const RepositoryContext = createContext({
  documentRepo: new TauriDocumentRepository(),
  highlightRepo: new TauriHighlightRepository(),
});

export function useDocumentRepository() {
  return useContext(RepositoryContext).documentRepo;
}
```

## Writing Tests

### Domain Unit Test (TypeScript)

```typescript
// src/__tests__/unit/highlight.test.ts
import { describe, it, expect } from 'vitest';
import { mergeOverlappingHighlights } from '../../domain/highlight-utils';

describe('mergeOverlappingHighlights', () => {
  it('merges adjacent highlights', () => {
    const highlights = [
      { rects: [{ x: 0, y: 0, width: 10, height: 10 }] },
      { rects: [{ x: 10, y: 0, width: 10, height: 10 }] },
    ];
    const merged = mergeOverlappingHighlights(highlights);
    expect(merged).toHaveLength(1);
  });
});
```

### Application Service Test with Mocks (Rust)

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use crate::ports::MockDocumentRepository;
    use mockall::predicate::*;

    #[tokio::test]
    async fn test_add_document() {
        let mut mock = MockDocumentRepository::new();
        mock.expect_add()
            .with(eq("/test.pdf".to_string()), eq(None))
            .returning(|path, _| Ok(Document { file_path: path, ..Default::default() }));

        let service = LibraryService::new(mock);
        let result = service.add_document("/test.pdf".to_string(), None).await;
        assert!(result.is_ok());
    }
}
```

### Architecture Test (ArchUnitTS)

```typescript
// src/__tests__/architecture/domain-boundaries.test.ts
import { describe, it, expect } from 'vitest';
import { projectFiles } from 'archunit';

describe('Domain Layer Boundaries', () => {
  it('domain should not depend on adapters', async () => {
    const rule = projectFiles()
      .inFolder('src/domain/**')
      .shouldNot()
      .dependOnFiles()
      .inFolder('src/adapters/**');
    await expect(rule.check({ allowEmptyTests: true })).resolves.toEqual([]);
  });
});
```

## Migration Workflow (Strangler Fig)

1. **Pick a module** (start with Settings - lowest risk)
2. **Define port trait** matching existing API
3. **Create application service** that uses the port
4. **Implement adapter** with existing logic
5. **Wire up in lib.rs** and verify identical behavior
6. **Add unit tests** for application service
7. **Add architecture tests** to enforce boundaries
8. **Migrate business logic** from adapter to service
9. **Remove legacy code** once stable
10. **Repeat** for next module

## Common Gotchas

1. **Don't import adapters in domain**: Domain layer is pure - no `@tauri-apps/api` imports
2. **Ports are interfaces, not implementations**: Don't put logic in port definitions
3. **Keep command handlers thin**: Parse input → call service → map output (~10 lines max)
4. **Attribute order matters**: `#[automock]` BEFORE `#[async_trait]`
5. **Use owned types**: `String` not `&str` in async trait methods (avoids lifetime issues)
6. **Regenerate types**: `pnpm tauri dev` triggers tauri-specta bindings generation
7. **Mock at port level**: Mock `MockDocumentRepository`, not `SqliteDocumentRepo`

## ESLint Boundary Rules

The ESLint configuration enforces these import rules:

| From | Allowed Imports |
|------|-----------------|
| `domain` | domain, ports |
| `ports` | ports |
| `application` | domain, ports, application |
| `adapters` | ports, domain, adapters |
| `components` | ports, domain, hooks, stores, lib, components |
| `hooks` | all except components |

Violations show as ESLint errors in IDE and CI.
